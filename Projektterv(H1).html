1. Fejlesztési terv: mérföldkövek, eredmények

A projekt fejlesztését szakaszokra bontjuk a hatékony munkavégzés érdekében. A Janus webalkalmazás célja egy modern, átlátható egyetemi adminisztrációs rendszer létrehozása, amely a hallgatók és oktatók számára egyaránt könnyen kezelhető.

  • 1. Mérföldkő (3. hét): Alapinfrastruktúra és Felhasználói Kezelés (Belépés, Regisztráció)
    • Eredmény: Működő, biztonságos bejelentkezési és regisztrációs felület. Beállított adatbázis (MySQL), amely képes kezelni a felhasználói (Hallgató, Oktató, Admin) profilokat.
  • 2. Mérföldkő (6. hét): Tanulmányi Modul Alapjai (Kurzusok, Órarend)
    • Eredmény: Oktatók számára kurzusok létrehozása és hallgatók hozzárendelése a kurzusokhoz. A hallgatók számára az aktuális órarend áttekinthető megjelenítése.
  • 3. Mérföldkő (9. hét): Pénzügyi és Kommunikációs Modul
    • Eredmény: Pénzügyi tételek (pl. félév díja, vizsgadíj) megjelenítése hallgatói felületen. Belső üzenetküldő rendszer (chat) megvalósítása a hallgató és oktató között.
  • 4. Mérföldkő (12. hét): Jegykezelés és Jelentések
    • Eredmény: Oktatók számára jegyek beírásának lehetősége a rendszerbe. Hallgatók számára a korábbi jegyek és a teljesítmény-átlagok lekérdezése. Belső hibakeresés és tesztelés befejezése.

2. Programozói nyelvek, fejlesztő eszközök

A projekt során a teljesítményt, a gyors fejlesztést és a könnyű karbantarthatóságot helyezzük előtérbe, ezért modern, ipari standard nyelveket és eszközöket választunk.

  • Backend:
    • Programozási nyelv: Java
    • Keretrendszer: Spring Boot – a gyors API fejlesztés és a vállalati szintű funkcionalitás miatt.
  • Frontend (Felhasználói felület):
    • Keretrendszer: Next.js – a gyors renderelés (SSR) és a kényelmes fejlesztés érdekében.
    • Nyelv: TypeScript – a kódminőség és a hibamentesség növelésére.
    • Stílusozás: Tailwind CSS és a hozzá tartozó HeroUI komponensek – gyors, modern és reszponzív felület létrehozására.
  • Adatbázis:
    • Adatbázis típusa: MySQL – megbízható és széles körben elterjedt relációs adatbázis.
  • Fejlesztői környezet:
    • IDE: JetBrains szoftverek (IntelliJ IDEA a Java backendhez, WebStorm a Next.js frontendhez) és Visual Studio Code.

3. Használni tervezett eszközök, technikák

A modern szoftverfejlesztési elvek betartására törekszünk.

  • Verziókövetés: Git – a forráskód megosztására, a változások nyomon követésére és a csapatmunka összehangolására. A dokumentáció a betyárbringantik nyilvános Git tárolóban lesz elhelyezve.
  • Fejlesztési Módszertan: Scrum vagy egyszerűsített Kanban – rövid sprint ciklusokkal, hogy rugalmasak maradjunk.
  • API Tervezés: RESTful architektúra – egységes, szabványos kommunikáció a frontend és a backend között JSON formátumú adatok felhasználásával.
  • ORM: Spring Data JPA a MySQL adatbázissal való egyszerű kommunikáció érdekében.
  • Dokumentáció: HTML dokumentumok

4. Leendő produktumok (dokumentum), és létrejöttük ideje

A fejlesztés során több dokumentumot is előállítunk, amelyek a projekt átláthatóságát és karbantarthatóságát biztosítják.

  • Projekt Terv:
    • Leírás: Meghatározza a projekt céljait, a csapatot, a fejlesztési fázisokat, a használt technológiákat és a verziókövetést.
    • Létrejött: A projekt kezdeti fázisában (1-2. hét).
  • Vízió dokumentum
    • Leírás: A webalkalmazásunk létrehozásának átfogó célja. A dokumentum tartalmazza a termék fő célját (modern, gyors Neptun-alternatíva), az üzleti lehetőségeket, a probléma megfogalmazását, a fő érintetteket, a végtermék jellemzőit és a minőségi elvárásokat.
    • Létrejött: 3. hét végéig.
  • SRS (Szoftver Követelmény Specifikáció)
    • Leírás: Részletesen és egyértelműen meghatározza a szoftver összes funkcionális (mit csináljon a rendszer, pl. "kurzusfelvétel") és nem funkcionális (hogyan működjön, pl. "teljesítmény, biztonság") követelményét.
    • Létrejött:
  • Analízis Modell
    • Leírás: Grafikus (UML) és szöveges modellekkel ábrázolja a rendszer belső felépítését és az adatok közötti kapcsolatokat, még a kódolás előtt.
    • Létrejött:
  • Rendszer Tervezés (Technikai Terv)
    • Leírás: Meghatározza a rendszer technikai architektúráját, a fő szoftverkomponenseket, az API-k felépítését, az adatbázis sémát (MySQL táblák), valamint a Spring Boot és Next.js modulok közötti kommunikációt.

5. Tesztelési követelmények

A minőség biztosítása érdekében a projekt során többféle tesztelést is elvégzünk.

  • Unit Tests:
    • Követelmény: Minden fontos funkció (pl. jelszó titkosítása, kredit számítás) rendelkezzen automatizált teszttel.
    • Technika: JUnit 5 (Java Spring Boot esetén), Jest (Next.js/TypeScript esetén).
  • Integrációs Tesztek:
    • Követelmény: Ellenőrizzük a MySQL adatbázissal való kommunikációt és az API végpontok megfelelő működését.
  • Felhasználói Elfogadási Tesztek:
    • Követelmény: A végfelhasználói szerepkörök bevonásával ellenőrizzük, hogy a rendszer megfelel-e a kezdeti elvárásoknak. Különös figyelmet fordítunk a HeroUI/Tailwind CSS alapú felület használhatóságára.
  • Biztonsági Tesztelés:
    • Követelmény: Különös figyelmet fordítunk a bejelentkezési adatok és a személyes adatok védelmére, elkerülve a gyakori biztonsági hibákat.
